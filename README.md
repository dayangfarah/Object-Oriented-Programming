Department: Software Engineering

Academic Session/Semester: 2024/2025-2

Credit Hours: 4

📖 Course Synopsis
This course presents the concepts of object orientation and object-oriented programming techniques using Java programming language. It provides students with a thorough look at the basic constructs of the Java programming language such as its basic data types and operations. It also emphasizes on the use of standard Java APIs that allow students to develop text-based and GUI applications. It will also provide the programming techniques on exception handling and input/output files. At the end of this course, students should be able to use the basic constructs in object-oriented programming and utilize the selected Java APIs.  

💭 Reflection
What I Learned
Enrolling in SECJ2154 reshaped how I decompose and resolve programmatic problems by shifting my logic away from linear procedural flows toward modular design ecosystems. Through the structural mechanics of the Java programming language, I learned to package system processes into dedicated object structures. Mastering core architectural tenets like encapsulation, inheritance relationships, and polymorphic execution behaviors taught me how to construct code assemblies that are intrinsically reusable and easy to scale.

Challenges & Growth
The learning curve deepened significantly when moving from elementary data variables to tracking complex structural dependencies between interrelated classes, such as composition and aggregation links. Conceptualizing abstract interfaces and handling checked versus unchecked runtime faults required an elevated style of system foresight. Navigating these constraints during laboratory tasks forced me to think systematically, translating intricate workflow conditions into precise visual blueprints via UML configuration models before writing a single line of runtime logic.

Future Application
The modular strategies and design patterns built into this Java curriculum provide a vital template for my long term approach to corporate software creation. I intend to use these defensive engineering blueprints to construct enterprise application components, configure thread safe execution loops, and isolate backend interfaces behind well defined API boundaries. Building a native intuition for structural object encapsulation prepares me to join complex team environments, ensuring my programmatic contributions remain easily legible, stable, and simple to test under production scale strain.
